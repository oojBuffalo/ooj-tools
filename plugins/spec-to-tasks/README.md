# spec-to-tasks

Turn a PRD, spec, or MVP document into a readable markdown task tree — entirely inside Claude Code.
No MCP server, no CLI daemon, no external API keys. Three skills cover the core breakdown loop:

| Skill | Trigger examples | What it does |
|-------|------------------|--------------|
| `parse-spec` | "break this PRD into tasks", "generate tasks from spec.md" | Reads a spec and writes one markdown file per top-level task, ordered and dependency-aware. |
| `analyze-complexity` | "analyze task complexity", "which tasks should I break down" | Scores each task 1–10, recommends subtask counts, and writes a complexity report. |
| `expand-task` | "expand task 3", "break the complex tasks into subtasks" | Decomposes a task into concrete `N.M` subtasks, using the complexity analysis as guidance when present. |

The skills are model-driven (Claude does the generation and writes the files). A small
zero-dependency helper, `scripts/tasks.py`, handles the deterministic bits: computing the next id,
rebuilding `tasks/overview.md`, and validating the dependency graph (missing deps, forward deps,
cycles).

## Task store

Everything lives in a plain `tasks/` directory at your project root — one markdown file per task,
plus generated `overview.md` and `complexity-report.md`. There is no JSON source of truth, so the
tree is easy to read, edit by hand, and track in git. The exact schema is documented in
[`references/task-format.md`](references/task-format.md).

Example:

```
tasks/
  overview.md               # generated dashboard
  complexity-report.md      # generated by analyze-complexity
  001-project-setup.md
  002-user-authentication.md
  ...
```

## Install

From the ooj-tools marketplace:

```
/plugin install spec-to-tasks@ooj-tools
```

Or try it in one session without installing:

```
claude --plugin-dir /path/to/ooj-tools/plugins/spec-to-tasks
```

Then just ask, e.g. *"break down docs/prd.md into tasks"*.

## Development
