# src

## Purpose
TypeScript source for the dox engine - the deterministic core that derives the doc tree, index, and drift report from the filesystem.

## Ownership
Parent: dox root. Owns the CLI implementation; compiles to `dist/` (the published `bin`).

## Local contracts (rules)
- ESM only (`"type": "module"`); intra-package imports use explicit `.js` extensions (NodeNext resolution).
- Deterministic jobs (scope/index/check) never call a model; only prose (J3) is model-written.
- The Child Docs Index region is owned by `dox sync` - never hand-edit between the dox index markers.
- `cli.ts` is the only entry point and the only file with a shebang; keep one command per `cmd*` function in `commands.ts`.

## Work guidance
- Module layout: `config` (constants/config/root), `scope` (dir/file scoping), `summary` (purpose extraction), `indexgen` (index block), `links`, `git`, `templates`, `commands` (the four subcommands), `cli` (arg parse + dispatch).
- Build with `npm run build`; the output in `dist/` is what hooks and `npx dox` run.

## Test & verify
- `npm run build` must succeed with no type errors, then `node dist/cli.js check` must report `dox: ok`.
- Behaviour parity is the contract: a change must not alter `check`/`sync`/`context` output for the dogfooded tree.
